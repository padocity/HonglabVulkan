동기화 수정 (2026-03-30)

  • 구형 30시리즈 GPU에서 오류가 제보되어 조사한 결과, 최신 권고사항이 반영되지 않은 부분을 확인하여 수정하였습니다.
  • 널리 사용되는 Sascha Willems 예제도 같은 문제가 있었으며 2025년 8월에 동기화 전체를 리워크했습니다.
  • Ex08 ~ Ex14 예제가 이 변경의 영향을 받습니다.

세부 변경 사항

  • 세마포어 인덱싱 수정 — 이전 프레임이 사용 중인 세마포어를 다음 프레임이 재사용할 수 있는 문제가 있었습니다. Khronos 권장 패턴 (2025-05)으로 변경.
    변경 전변경 후
    이미지 획득presentSemaphores_[currentSemaphore]imageAcquiredSemaphores_[currentFrame]
    렌더 완료renderSemaphores_[currentSemaphore]renderDoneSemaphores_[imageIndex]
  • 변수명 변경 — 세마포어의 실제 역할을 알기 쉽게 이름을 바꿨습니다.
    변경 전변경 후역할
    presentSemaphores_imageAcquiredSemaphores_이미지 획득 완료 시 signal
    renderSemaphores_renderDoneSemaphores_렌더링 완료 시 signal
  • MSAA 레이아웃 전환 추가 — MSAA 렌더 타겟이 초기 레이아웃 전환 없이 사용되고 있었습니다. 최신 SDK에서 오류가 발생할 수 있습니다.
  • Windows CMake 빌드 지원 — cmake --preset vs2022-vcpkg && cmake --build --preset debug
